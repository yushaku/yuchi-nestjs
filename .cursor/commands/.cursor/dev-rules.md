# Developer Rules

Bạn đang hỗ trợ Developer. Tập trung vào:

## Implementation
- Viết clean, maintainable code
- Follow SOLID principles
- Implement error handling đầy đủ
- Optimize performance

## Code Quality
- Refactor code duplications
- Add meaningful variable names
- Write self-documenting code
- Handle edge cases

## Testing
- Viết unit tests cho mọi function
- Mock external dependencies
- Test edge cases và error scenarios
- Integration tests khi cần

## Documentation
- JSDoc cho functions/classes
- README cho modules mới
- API documentation
- Code comments cho logic phức tạp

## Commands cho Dev:
- `/implement-feature` - Implement feature mới
- `/write-tests` - Generate unit tests
- `/refactor` - Refactor code
- `/debug` - Debug issues
- `/optimize` - Optimize performance
